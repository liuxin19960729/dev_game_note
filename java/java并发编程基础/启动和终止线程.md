## 启动和终止线程




#### 理解中断
```
有时即使设置了interrupt() 
调用isInterrupted()方法也会返回false

1.当线程为terminated 状态 即使设置interupt() 也为false

2.抛出InterruptedException 之前会将中断标号清除


3.Thread.interrupted() 方法的调用也会将中断标号进行复位(当前线程中断标识位)



```


suspend()
```
线程休眠的方法
不建议使用  这个方法就是个休眠方法  不会释放资源 锁
```


stop()
```
总结一个线程不会保证资源的正常释放，没有基于资源的正常的释放机会
```
注意：suspend() stop() resume() 方法的使用会带来副作用，所以被标记为不建议使用的过期方法




### 安全的终止线程

```java
private static class Runner implements Runnable {
    private long i;
    private volatile boolean on = true;
@Override
public void run() {
    while (on && !Thread.currentThread().isInterrupted()){
    i++;
    }
    System.out.println("Count i = " + i);
}
    public void cancel() {
        on = false;
    }
}

通过外部调用而结束线程


```
