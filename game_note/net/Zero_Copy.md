# Zero Copy技术
```
现带网络应用的性能瓶颈:I/0

传统的Linux io 接口 数据拷贝操作
    什么是数据拷贝操作？
        1.os kernel 缓冲区------>application 缓冲区  之间进行传输
        
        设置缓冲好处：减少disk io  
            当数据未放在缓冲区-->访问数据就要进行 磁盘 io-->会依赖(深度)cpu去操作 -->拷贝数据的时候会极大依赖cpu-->导致极大的系统的开销-->这样会导致传输效率非常慢

    io效率低下，这样有会影响网路服务器性能地下-->根据木桶规则，io是他的短板(瓶颈)-->解决了这个问题-->网络服务器性能也会提高

```
